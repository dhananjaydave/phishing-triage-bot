Phishing Triage Bot

Send a .eml to a private Telegram bot, get back a verdict plus the full evidence behind it — not just a label you have to trust blindly.

Live demo

Limited to 5 analyses/hour per visitor. Nothing submitted here is stored - it's parsed in memory and discarded after the report is returned. Don't paste real victim/PII data if you don't have to.

    Example report

    🚨 LIKELY PHISHING Subject: Urgent: Your account will be suspended - verify now From: PayPal Security <support@paypa1-secure.tk> Evidence: 🔴 [authentication] SPF/DMARC failed - the sending server isn't authorized to send as this domain. 🔴 [sender_spoofing] Display name claims to be 'PayPal Security', but the actual sending domain is 'paypa1-secure.tk', not paypal.com. 🟡 [reply_to] Reply-To domain differs from the From domain - replies get redirected elsewhere. 🟡 [social_engineering] Urgency/pressure language found: urgent, act now, immediately. 🔴 [suspicious_url] Link uses a raw IP address instead of a domain. 🔴 [link_mismatch] Link text 'https://paypal.com/verify' doesn't match where it actually points. 🔴 [attachment] Attachment 'invoice.exe' has a dangerous extension.

    What it checks

    AuthenticationSPF / DKIM / DMARC results
    Brand impersonationdisplay name vs. actual sending domain
    Reply-To mismatchreplies redirected elsewhere
    Social engineeringurgency / pressure language
    Suspicious URLsraw IPs, shorteners, abused TLDs, lookalike domains
    Link mismatchvisible text vs. actual href
    Attachmentsdangerous extensions
    IOC enrichmentevery domain/IP/hash checked against a separate enrichment API

    Architecture

    • 01
      Parse. Stdlib email package only - headers, body text/HTML, every link's (visible text, href) pair, every attachment's SHA256.
    • 02
      Score. Seven independent heuristic checks, each returning evidence at malicious/suspicious/info severity - including the "looks fine" results, not just red flags.
    • 03
      Enrich. Every extracted indicator gets checked against the IOC Enrichment API; degrades gracefully if that's unreachable.
    • 04
      Verdict. Any malicious signal anywhere -> phishing. Any suspicious signal short of that -> suspicious. Deliberately simple and auditable, not a tuned score nobody can explain.
